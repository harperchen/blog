Usages and problems of Taint Tools

Posted on 2020-09-15 Edited on 2020-09-21 In Taint

Existing taint analysis tools can be divided into three categories.

  • Dynamic Taint Analysis: The first category of tools track the information flow from taint source to taint sink following the runtime execution trace, given a specific input. Most of these dynamic analysis tools are built on the top of dynamic binary instrumentation (DBI) framework such as Pin and Valgrind. Some other tools like PolyTracker works as a LLVM Pass that instruments taint tracking logic into the programs during compilation to perform taint analysis with lower overhead.

    Tools DBI Platform or LLVM Pass Data Dependency
    Dytan DBI Platform: Pin Explicit and Implicit
    Triton DBI Platform: Pin Explicit
    LibDFT DBI Platform: Pin Explicit
    LibDFT64 DBI Platform: Pin
    Taintgrind DBI Platform: Valgrind Explicit
    DECAF/DECAF++ DBI Platform: Qemu
    PolyTracker LLVM Pass started from DataFlowSanitizer (DFSan) Explicit

  • The second category of tools work as interpreter or cpu emulators, which means these tools not really execute the target program, but emulating the functionalities and collect the essential information for analysis. Usually we need to setup the virual stack, heap, memory, and virtual input before performing analysis. These tools process machine code one by one and transform the assembly to their intermediate language for memory and register manipulation.

    Tools Data Dependency
    Bincat (Plugin for IDA)
    Triton

  • Static Taint Analysis: The third category of tools analyzes the target program statically and wholly. We need ti specify the taint source and sink

    Tools Data Dependency
    Pyre-check static analysis tool reasons about data flows in Python applications
    Psalm static analysis tool for finding errors in PHP applications
